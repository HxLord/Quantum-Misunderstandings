# HELIOS GUARD
---
## Project Description

**Helios Guard** - это MVP-система защиты спутников от солнечных бурь и космической погоды. Система реализует полный цикл обнаружения, анализа и защитого маневра.


### Назначение

Система предназначена для демонстрации концепции TinyML-based защита спутников:
- Мониторинг солнечной активности в реальном времени
- Раннее обнаружение опасных событий (вспышки Х-Класса)
- Расчет времени прибытия частиц (ТоА)
- Автоматическое выполнение защитных команд

### Ключевые возможности

| Возможность | Описание |
|-------------|----------|
| SensorUnit | Генерация/чтение данных рентгеновского потока |
| Tiny Trigger | Низкопотребляющий пороговый детектор |
| MissionAI | Расчет ТоА и команды защиты |
| Energy Simulation | Моделирование энергопотребления |
| Real-time Visualization | График активности в Matplotlib |

# Симуляции для понятности принципа работы продукта:

## 1. Симуляция работы HELIOS-GUARD на базе: CesiumJS Sandcastle

Система интерактивного мониторинга и автоматического реагирование спутника на солнечную активность в реальном времени.

## Обзор 

Проект представляет собой 3D-симулятор, который можно использовать для мониторинга солнечной активности в реальном времени с помощью анализа данных от **NASA DONKI API**. Система классифицирует солнечные вспышки (от класса А до Х) и автоматически изменяет режим работы бортовых систем для обеспечения выживаниемости аппарата.

## Основные возможности системы

* **Real-time Data:** Опрос серверов Nasa на предмет текущей солнечной погоды.
* **Automated Logic:** Алгоритм классификации угроз и принятия решений (Safe Mode, Alert, Nominal).
* **Visual HUD:** Футуристичный интерфейс на HTML/CSS для мониторинга систем спутника.
* **3D Visualization:** реалистичная отрисовка орбиты и каналов связи с землой на движке CesiumJS.

## Стек технологий

* **CesiumJS** — 3D движок для визуализации космического пространства.
* **NASA DONKI API** — источник реальных данных о космической погоде от NASA.
* **JavaScript (ES6+)** — логика обработки и интеграции данных.
* **HTML5/CSS3** — графическая оболочка интерфейса (HUD).

## Классификация действий системы

| Класс вспышки | Уровень угрозы | Реакция системы | Состояние связи |
|:---:|:---:|:---|:---:|
| **A/B** | Низкий | Штатный сбор данных | Стабильно |
| **C** | Умеренный | Логирование аномалий | Стабильно |
| **M** | Высокий | Переход на резервные системы | Нестабильно |
| **X** | Критический | **SAFE MODE**: Защитные экраны ВКЛ | Обрыв связи |

# 2. Обьективная симуляция проекта на базе

Интерактивная модель на Python (Pygame + Matplotlib), демонстрирующая воздействие космической радиации на орбитальные аппараты и эффективность систем автоматизированной защиты.

## Основные возможности
* **Визуализация частиц**: Моделирование солнечного ветра (классы A-X) и альбедо-излучения Земли.
* **Интеллектуальная защита**: Автоматический переход спутников в режим `SHIELD` при обнаружении частиц высокой энергии.
* **Мониторинг в реальном времени**: Логарифмический график потока рентгеновского излучения с отметками срабатывания систем защиты.
* **Интерактивность**: Управление скоростью симуляции и возможность ручного сброса (Reload).

## Установка и запуск
1. Установите зависимости:
```bash
pip install pygame matplotlib
```
2. Запустите симуляцию:
```bash
python main.py
```
## Управление
`SPACE` — Включить/выключить систему автоматической защиты.

`R` — Полный сброс симуляции (Reload).

`Slider` — Изменение скорости протекания физических процессов.

# 3. Helios Guard: AI-Driven Space Weather Shield. Система ИИ (продукт) 

**Helios Guard** — это бортовая система предсказания солнечных вспышек на базе TinyML для CubeSat. Система анализирует потоки рентгеновского и протонного излучения в реальном времени, обеспечивая запас времени (Lead Time) до 12 часов для активации защитных маневров.

## Основные характеристики

* **Архитектура:** 1D-CNN (Convolutional Neural Network), оптимизированная для Edge AI.
* **Модель:** TensorFlow Lite (INT8 Quantized), вес ~45 КБ.
* **Целевое железо:** ARM Cortex-M7 (STM32H7 / Teensy 4.1).
* **Эффективность:** Среднее время упреждения — 12 часов при False Positive Rate < 0.2%.

## Быстрый старт (Google Colab / Local)

Для запуска симуляции обучения на синтетическом датасете:
1. Клонируйте репозиторий.
2. Установите зависимости:
```bash
pip install tensorflow numpy matplotlib scikit-learn
```
3. Запустите скрипт:
```bash
python train_model.py --quick
```
